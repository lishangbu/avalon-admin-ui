# Playwright 测试模式

## Mock 模式

用于默认完整前端验证。通过受控路由响应覆盖菜单 smoke、资料 CRUD、战斗规则和沙盒流程；响应必须保持真实 OpenAPI 形态。

## 真实后端模式

只在任务明确涉及联调、认证服务器、数据库或运行时战斗行为时启用。显式设置仓库约定的环境变量并启动后端；不得静默回退到 mock。

## 选择标准

- 页面内部逻辑：Vitest/Testing Library。
- 多页面、路由、菜单和浏览器行为：Mock Playwright。
- 前后端协议或运行时集成：真实后端 Playwright。

报告每种模式的实际运行结果；skip 不等于通过。
